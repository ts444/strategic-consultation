## Codebase Patterns
- Python package root is `/home/tim/strategic-consultation/`; run pytest/ruff/mypy from there
- `harness/` is a Python package (has `__init__.py`); tests live in `harness/tests/`
- `pyproject.toml` at repo root configures ruff (target py310), mypy (strict), pytest testpaths
- `_read_harness_version()` in harness/init.py resolves CHANGELOG.md via `Path(__file__).parent.parent`
- prd.json lives at `.ralph/prd.json`, not at the repo root
- progress.txt lives at the repo root (same dir as CLAUDE.md and design-principles.md)
- All markdown documents (templates, schemas) reference design-principles.md as the binding spec
- Worked examples belong inline inside schema docs, not in separate fixtures
- Claim ids use block-type prefix + NNN (GAP-NNN, REC-NNN, RSK-NNN, RMI-NNN, BUD-NNN, CMP-NNN)
- MSP portfolio lives at `~/msp-portfolio/` (outside the harness repo); commit service changes there separately
- Portfolio MCP reads from `~/msp-portfolio/` via PORTFOLIO_ROOT = Path("~/msp-portfolio").expanduser()
- Service files use YAML frontmatter (name, domain, tags, pricing tiers, SLA tiers) + markdown body sections
- Orchestrator prompts live at `prompts/orchestrator/NN-phase.md`; HITL gate checklists are embedded in those files
- Validator enforces block schemas; update both the validator rules and relevant orchestrator prompt schema docs together
- Change request files live at `backlog/change-requests/CR-YYYY-NNN.md`; update `status:` field to `shipped` when resolved
- pyproject.toml had no `[project]` section prior to US-036; added it with `[build-system]` and `[project].dependencies`
