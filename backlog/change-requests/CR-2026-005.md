# CR-2026-005: Intake template — add budget-validation prompt before envelope is fixed

**Originating engagement:** Meridian-Logistics
**Observed in phase:** 00-intake

## Problem

The declared BUD-001 budget envelope (capex_y1 £50k, opex_annual £20k, hard limit £200k total) was 25× below the programme cost identified in phase 04-roadmap (≥£492,480/yr in Silver-tier opex alone, exceeding the £200k total hard limit in a single year).

The intake scope template prompted for the customer's stated budget envelope but did not include any step to sense-check whether the declared envelope was consistent with the likely programme cost for a NIS2/TISAX-driven engagement at the customer's headcount. The mismatch was only discovered in phase 04-roadmap, at which point it became a blocking open question (OQ-P04-001) that the handover had to surface as a top risk to the CEO.

## Proposed change

Add a budget-validation prompt section to `templates/scope.template.md` (§3.1 Budget Envelope) that:

1. Asks the consultant to estimate the approximate number of new services likely required before fixing the envelope.
2. Provides a worked example of Silver-tier per-user-per-month costs for a typical NIS2/TISAX programme at common headcount bands.
3. Explicitly prompts the consultant to flag if the declared envelope is likely below programme cost before ratification, and to capture this as an open question for the customer if so.

## Expected improvement

Budget misalignment is surfaced at intake (phase 00) rather than phase 04, allowing the customer to recalibrate expectations before the engagement is fully scoped. Reduces the likelihood that a roadmap is ratified against an unrealistic envelope, and removes a common top-of-list handover risk.
