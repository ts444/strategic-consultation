# CR-2026-001: Validator body_lines() should skip HTML comment blocks

## Problem

During the Meridian Logistics engagement, `00-intake/scope.md` was drafted from `scope.template.md`. The template's HTML comment blocks (`<!-- ... -->`) contain illustrative claim-annotation examples — source URI shapes, label names, confidence markers — embedded as documentation for authors.

The validator utility `body_lines()` in `validator/_util.py` does not skip lines inside HTML comment blocks. As a result, any artifact drafted directly from a template triggers `claim_label_present` and `label_producer_binding` violations because the comment-block content is treated as live claim content.

This is a class of false positive that affects every artifact produced from any template that includes illustrative examples in comments. It adds a mandatory manual step (stripping all comment blocks before ratifying) that is nowhere documented and creates confusion for first-time users of each template.

## Proposed Change

Update `validator/_util.py` `body_lines()` to track whether each line falls inside an HTML comment block (`<!-- ... -->`). Lines inside a comment block should be yielded with a flag or simply skipped, consistent with how markdown processors treat HTML comments as non-content.

Specifically:
- Track `in_comment` state as a boolean toggled by `<!--` and `-->` markers.
- Skip (do not yield) any line between those markers.
- Handle multi-line comments (opening `<!--` and closing `-->` on different lines).
- Handle single-line comments (`<!-- ... -->` on one line).

## Expected Improvement

Once implemented, template comment blocks are no longer a source of false positives. Authors can draft artifacts from templates and run the validator immediately, without first manually stripping all comment content. The validator becomes safe to use on template files themselves.

## Scope

- `validator/_util.py` — update `body_lines()`
- `validator/tests/test_rules.py` — add fixture covering artifact with HTML comment block containing claim-label patterns; assert zero violations
